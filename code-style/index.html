# Code Style

曾经我们定义一套 ESlint 的规则来约束代码规范，但是使用的过程并不让人愉快 💩 ，就像公司一些值得商榷的规定，却需要我们花一定的时间去应对，直到我们使用 Prettier 来进行代码规范的约束：

  • 你不必再去了解 ESlint 的配置
  • 不需要再看到代码不符合规范时那红色、绿色的波浪下划线
  • 只需要简单配置，即可在保存时自动格式化

从此写代码愉悦感提升 💯

# VSCode 配置

安装 prettier-vscode 插件， 在 VSCode 设置中增加如下配置，即可在 *.js*.ts*.vue 文件中为相应的代码进行自动格式化。

{
    "[javascript]": {
        "editor.formatOnSave": true
    },
    "[typescript]": {
        "editor.formatOnSave": true
    },
    "[vue]": {
        "editor.formatOnSave": true
    },
    "vetur.format.defaultFormatter.js": "prettier",
    "vetur.format.defaultFormatter.ts": "prettier",
    "vetur.format.defaultFormatter.css": "prettier"
}

因为历史原因我们增加一些规范修改 .prettierrc:

{
    "tabWidth": 4,
    "singleQuote": true
}

更详细的说明，请查看 Prettier。
